# Decision Rules

- Sempre ler o card antes de qualquer plano de escrita.
- Nunca adicionar tag ou mover card sem `--execute`.
- Se nao houver CPF, proposta ou request id, pedir informacoes minimas.
- Se houver CPF/proposta, marcar os checks operacionais como prontos para execucao.
- Separar fatos do card, inferencias e recomendacoes.
