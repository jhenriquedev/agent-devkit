# Knowledge

Conhecimento operacional do BPO Analyser.

- `system.md`: identidade, escopo, principios de decisao e guardrails do agente.
- `context.md`: origem dos contratos BPO e proibicao de API SelfHire.
- `policies.yaml`: politicas de seguranca, integracao, decisao e escalacao.
- `domain-glossary.md`: dicionario de situacoes BPO, regra Core de elegibilidade,
  CPF e documentos anexados.
- `prompts/`: prompts agentic especificos por capability.

Endpoints opcionais de esteira, formalizacao e consignacao aparecem no
diagnostico de configuracao, mas nao possuem capability operacional nesta spec.
