# Regras

- Criar template apenas quando o usuario nao fornecer um template existente adequado.
- Perguntar antes de salvar template reutilizavel no projeto.
- Incluir abas, campos obrigatorios, validacoes e exemplos minimos coerentes com o brief.
- Evitar regras de negocio implicitas; registrar gaps, premissas e duvidas.
- Gerar `.xlsx` como artefato principal e preservar rastreabilidade do brief.
- Planejar versionamento antes de registrar o template.
