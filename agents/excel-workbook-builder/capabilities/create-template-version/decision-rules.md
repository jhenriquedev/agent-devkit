# Regras

- Sempre criar nova versao; nunca sobrescrever versao validada.
- Usar `base_version` ou template recebido como fonte rastreavel.
- Registrar alteracoes estruturais, formulas, validacoes e impacto esperado.
- Validar formulas e ranges chave antes de recomendar promocao.
- Manter compatibilidade de schema ou documentar quebra explicitamente.
- Nao promover automaticamente a nova versao.
