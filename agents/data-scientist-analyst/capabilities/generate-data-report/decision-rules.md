# Regras

- Gerar relatorio markdown somente quando `output` for informado.
- Incluir fonte, hash, qualidade, perfil de colunas, dados sensiveis e regras usadas.
- Mascarar dados pessoais e evitar despejo de amostras grandes.
- Separar fatos do dataset de hipoteses sobre causa de problemas.
- Declarar limitacoes de formato, leitura, linhas ignoradas ou truncamento.
- Nao alterar a fonte nem publicar relatorio em sistema externo.
