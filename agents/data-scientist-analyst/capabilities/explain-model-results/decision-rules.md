# Regras

- Explicar driver principal, regra selecionada, metricas e limitacoes do baseline.
- Nao atribuir importancia causal a uma feature sem desenho apropriado.
- Declarar quando resultado for in-sample ou heuristico.
- Sinalizar leakage, desbalanceamento e baixa cobertura como riscos.
- Usar linguagem compreensivel para publico tecnico e executivo.
- Nao recomendar deploy ou automacao sem avaliacao adicional.
