# Regras

- Criar ou planejar design web responsivo com breakpoints, estados e handoff.
- Cobrir estados empty, loading, error, success e permission nas telas principais.
- Validar hierarquia, foco visivel, contraste e labels de formulario.
- Reutilizar design system existente ou declarar tokens e componentes propostos.
- Escrita Figma exige confirmacao, direct mode e evidencia do bridge.
- Sem bridge, gerar plano executavel, screen map, design-system-spec e dev-handoff.
