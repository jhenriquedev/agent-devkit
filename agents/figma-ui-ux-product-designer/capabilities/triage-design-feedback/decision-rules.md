# Regras

- Classificar feedback em Visual, UX, Negocio, Acessibilidade ou Handoff.
- Priorizar critico, alto, medio ou baixo conforme impacto.
- Feedback de negocio exige pergunta aberta antes de alterar design.
- Feedback de acessibilidade tem prioridade minima alta.
- Registrar conflitos entre feedbacks em `open-design-questions.md`.
- Nao aplicar mudancas nesta capability; gerar triagem acionavel.
