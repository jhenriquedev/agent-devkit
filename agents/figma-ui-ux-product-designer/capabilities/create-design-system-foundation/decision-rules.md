# Regras

- Buscar libraries, componentes, variaveis e estilos existentes antes de criar do zero.
- Definir tokens de cor, tipografia, spacing, radius e componentes base com variantes e estados.
- Incluir estados de interacao, foco, erro, disabled e loading quando aplicavel.
- Validar contraste e acessibilidade basica nas foundations.
- Escrita em Figma exige confirmacao e evidencia do bridge; sem isso, gerar `design-system-spec.md`.
- Nao sobrescrever design system validado sem criar versao ou proposta separada.
