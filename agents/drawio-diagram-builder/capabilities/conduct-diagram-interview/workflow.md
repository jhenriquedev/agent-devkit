# Workflow: Conduct Diagram Interview

OBJETIVO: Fechar apenas as lacunas bloqueantes para diagramar, com o mínimo de
perguntas.

ENTRADAS: brief, diagram_type, audience, detail_level e (se houver) source-context.

RACIOCÍNIO:
1. Leia o material já disponível. Marque quais das dimensões obrigatórias de
   policies.yaml (objetivo, audiencia, tipo_de_diagrama, nivel_de_detalhe, fontes,
   escopo, fora_de_escopo, elementos_obrigatorios) JÁ estão respondidas.
2. Pergunte SÓ as que faltam. Nunca pergunte algo já presente no material.
3. Agrupe por: objetivo/decisão, escopo, conteúdo obrigatório, audiência, forma visual.
4. Se a demanda for pequena e clara, faça no máximo 2 perguntas ou nenhuma.

DECISÃO: Se 0 lacunas bloqueantes → declare "pronto para planejar" e não pergunte.

SAÍDA: diagram-interview.md com seções Perguntas (só as necessárias) e
Confirmações (o que já foi entendido).

NÃO FAZER: despejar questionário fixo; perguntar audiência se a fonte já a define.
