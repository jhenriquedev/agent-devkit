# Visual Rules

- Use grupos por camada, ator, sistema ou dominio quando houver mais de cinco
  elementos.
- Prefira leitura da esquerda para a direita para fluxos de negocio e de cima
  para baixo para hierarquias.
- Use cores com significado estavel, nao decoracao.
- Rotule setas que carregam acao, evento, decisao, protocolo ou payload.
- Mantenha labels curtos dentro das caixas e detalhe complementar em notas.
- Inclua legenda quando houver cor, tipo de linha ou shape com significado.
- Se o diagrama tiver muitas responsabilidades, recomende separar em paginas ou
  diagramas menores.
