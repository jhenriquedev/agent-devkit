# Workflow

1. Carregar spec YAML/JSON.
2. Validar alvo desktop, janela, passos, pre-condicoes e estrategia de
   verificacao.
3. Confirmar que alternativas mais seguras foram avaliadas.
4. Bloquear quando existir API, CLI, MCP, Playwright, Selenium ou automacao
   nativa aplicavel.
5. Retornar plano read-only com riscos, guardrails e artefatos previstos.
