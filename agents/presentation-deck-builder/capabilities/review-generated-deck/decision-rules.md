# Decision Rules

- Revisar deck gerado sem modificar o arquivo original.
- Validar aderencia ao template, campos obrigatorios, placeholders vazios, overflow e consistencia visual.
- Checar se o conteudo entregue corresponde ao input estruturado e ao plano aprovado.
- Nao aprovar deck com dados inventados ou campos obrigatorios ausentes.
- Separar problemas bloqueantes de ajustes recomendados.
- Registrar slide, layout e placeholder afetados quando possivel.
- Considerar slides superlotados, hierarquia visual quebrada e imagens ausentes como riscos de qualidade.
- Nao alterar catalogo, template ou versao durante a revisao.
- Se nao houver template de referencia, declarar limite da revisao.
- A saida deve orientar `refine-generated-deck` ou nova geracao.
