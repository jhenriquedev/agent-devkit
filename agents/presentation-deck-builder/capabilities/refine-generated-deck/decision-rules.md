# Decision Rules

- Aplicar feedback ao deck gerado sem alterar o template base.
- Preservar layouts, identidade visual, placeholders e restricoes de overflow.
- Se o feedback exigir mudanca de template, encaminhar para `refine-template` e nova versao.
- Nao inventar conteudo ausente para satisfazer feedback ambiguo.
- Perguntar quando feedback conflitar com template, dados de entrada ou objetivo do deck.
- Gerar novo arquivo de saida; nao sobrescrever entrega anterior sem confirmacao.
- Revalidar campos obrigatorios, placeholders vazios, texto excedente e consistencia visual.
- Manter rastreabilidade entre feedback, slides alterados e resultado.
- Nao modificar catalogo ou manifests.
- A saida deve poder ser revisada por `review-generated-deck`.
