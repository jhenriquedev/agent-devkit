# Decision Rules

- Criar template somente apos obter brief suficiente sobre objetivo, publico, estilo, identidade visual e tipos de slide.
- Perguntar antes de salvar qualquer template em `templates/`.
- Registrar novo template como ativo versionado, nao como arquivo descartavel.
- Marcar template novo como `draft` ate aprovacao explicita.
- Criar manifest, versao inicial, input schema, slide-map, usage notes e changelog.
- Nao inventar identidade visual, logos ou conteudo de negocio ausente.
- Preservar compatibilidade de paths e nomes em kebab-case.
- Definir placeholders claros para dados variaveis em vez de texto fixo ambíguo.
- Nao sobrescrever template existente com o mesmo id sem decisao explicita.
- A saida deve deixar claro o que falta para validar e reutilizar o template.
