# Decision Rules: Generate Incident Report

- Separar sumario, timeline, evidencias, hipoteses, lacunas e proximos passos.
- Timeline deve ser ordenada por timestamp e conter somente eventos representativos.
- Causa raiz deve permanecer hipotese se nao houver evidencia suficiente.
- Nao afirmar impacto de negocio sem informacao externa aos logs.
- Declarar explicitamente que nenhuma escrita foi executada.
- Resumir mensagens grandes e preservar dados sensiveis.
