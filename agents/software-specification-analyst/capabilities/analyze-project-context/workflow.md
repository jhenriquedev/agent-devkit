# Workflow: Analyze Project Context

1. Validar o caminho do projeto.
2. Classificar ou receber profundidade: `light`, `medium` ou `deep`.
3. Mapear documentacao, estrutura, dependencias, codigo, testes, configuracoes e
   integracoes aparentes.
4. Separar fatos observados no codigo de inferencias e regras confirmadas.
5. Gerar documentos intermediarios de analise.
6. Levantar perguntas de negocio a partir das lacunas encontradas.
7. Encerrar com recomendacao de proximo passo: entrevista, revisao ou
   especificacao final.
