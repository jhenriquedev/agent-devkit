# Decision Rules

- Gerar spec final somente a partir de analise madura e contexto suficiente.
- Verificar todos os criterios de contexto suficiente de `knowledge/policies.yaml`.
- Separar decisoes confirmadas de pendencias e perguntas abertas.
- Se houver pergunta bloqueante, produzir artefatos de pendencia e nao declarar handoff pronto.
- Preservar rastreabilidade entre discovery, analise, decisoes e requisitos finais.
- Incluir as secoes obrigatorias da especificacao completa quando aplicavel.
- Nao inventar regra de negocio para preencher lacuna da analise.
- Gerar `open-questions.md` quando existirem questoes remanescentes.
- Pedir confirmacao antes de criar diretorio de saida ou sobrescrever arquivos.
- A saida deve preparar handoff para desenvolvimento com riscos e dependencias claros.
