# Decision Rules

- Ler analise existente e feedback antes de alterar qualquer conclusao.
- Incorporar respostas como fatos confirmados somente quando o feedback for claro.
- Preservar historico de decisoes relevantes em `decision-log.md`.
- Separar perguntas resolvidas de perguntas ainda abertas.
- Nao apagar contradicoes sem registrar como foram resolvidas.
- Ajustar premissas e inferencias afetadas pelo feedback.
- Se feedback criar nova lacuna bloqueante, impedir spec final pronta.
- Gerar artefatos refinados sem sobrescrever sem confirmacao.
- Manter rastreabilidade entre feedback, documentos alterados e decisoes.
- Pedir confirmacao antes de criar diretorio de saida.
