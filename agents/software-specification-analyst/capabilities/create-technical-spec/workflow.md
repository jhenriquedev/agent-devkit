# Workflow: Create Technical Spec

1. Extrair capacidades tecnicas implicadas pelos requisitos.
2. Identificar componentes, interfaces, dados, integracoes e estados.
3. Marcar decisoes arquiteturais em aberto.
4. Documentar seguranca, observabilidade, rollout e rollback.
5. Definir estrategia de testes e qualidade.
6. Separar recomendacoes tecnicas de requisitos confirmados.
