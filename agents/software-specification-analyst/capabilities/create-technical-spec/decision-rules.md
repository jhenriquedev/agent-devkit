# Decision Rules

- Derivar arquitetura, componentes e integracoes a partir de requisitos confirmados e contexto tecnico.
- Separar recomendacao tecnica de requisito tecnico obrigatorio.
- Documentar dados, APIs, estados, seguranca, permissoes, observabilidade, rollout e rollback.
- Nao escolher tecnologia, servico ou arquitetura sem base no contexto ou decisao registrada.
- Marcar decisoes arquiteturais abertas com impacto e opcoes conhecidas.
- Diferenciar fato observado no codigo de diretriz tecnica futura.
- Incluir estrategia de testes e qualidade alinhada aos requisitos.
- Registrar riscos, dependencias externas e restricoes operacionais.
- Nao propor mutacao em projeto ou implementacao nesta capability.
- A saida deve ser rastreavel para requisitos funcionais e nao funcionais.
