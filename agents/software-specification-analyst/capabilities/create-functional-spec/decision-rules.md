# Decision Rules

- Identificar problema, objetivo, atores, superficies e fronteiras de escopo antes dos requisitos.
- Separar requisitos funcionais, regras de negocio, criterios de aceite e recomendacoes.
- Numerar requisitos funcionais de forma estavel.
- Mapear fluxos principais, alternativos e excecoes.
- Nao inventar ator, permissao, regra ou comportamento ausente.
- Registrar premissas e perguntas abertas junto ao requisito afetado.
- Diferenciar fora de escopo de requisito pendente.
- Validar criterio de aceite como comportamento observavel.
- Usar Mermaid quando o fluxo precisar ser representado visualmente.
- A saida deve alimentar historias, rastreabilidade e spec tecnica.
