# Decision Rules

- Consolidar discovery notes, analise de projeto, perguntas, pontos criticos e decisoes abertas.
- Remover duplicidades sem apagar contradicoes relevantes.
- Separar fatos, inferencias, premissas, decisoes confirmadas e perguntas abertas.
- Destacar perguntas bloqueantes antes de recomendar especificacao final.
- Avaliar os criterios de contexto suficiente definidos em `knowledge/policies.yaml`.
- Nao promover regra implicita de codigo a regra de negocio confirmada.
- Registrar riscos, dependencias e escopo ainda incerto.
- Manter referencias de origem para cada conclusao importante.
- Se contexto for insuficiente, entregar dossier com perguntas e parar.
- A saida deve orientar refinamento, entrevista ou spec final.
