# Decision Rules: Generate Technical Docs

- Markdown e a fonte canonica.
- PDF deve ser derivado do mesmo conteudo.
- Separar explicitamente evidencias, inferencias e perguntas pendentes.
- Documentar contrato, ambientes, autenticacao, operacoes, payloads, erros, fluxos, testes e lacunas.
- Preservar evidencia de origem para trechos inferidos de texto livre.
- Nao publicar tokens, API keys, senhas, cookies ou `Authorization` completo.
- Incluir ordem de uso e pre-condicoes antes de exemplos executaveis.
- Diferenciar exemplo documentado de massa sintética.
- Quando PDF for gerado, manter paridade com o Markdown canonico.
- Incluir limitações por protocolo e artefatos equivalentes quando Postman nao for adequado.
- Marcar blockers que impedem teste real ou mutation segura.
